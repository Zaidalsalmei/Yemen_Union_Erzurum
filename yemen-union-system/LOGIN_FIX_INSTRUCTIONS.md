# تعليمات إصلاح تسجيل الدخول
## Yemen Student Union System - Login Fix Instructions

### المشكلة التي تم حلها
كان هناك خطأ في ملف `reset_admin.php` حيث كان يحاول استخدام كلاس `App\Core\Database` بدلاً من `App\Config\Database` الصحيح.

### التغييرات التي تم إجراؤها

#### 1. تحديث ملف `reset_admin.php`
- تم إعادة كتابة السكريبت بشكل كامل لضمان:
  - تحميل autoloader بشكل صحيح
  - استخدام الـ namespace الصحيح `App\Config\Database`
  - معالجة أفضل للأخطاء
  - رسائل واضحة ومفيدة للمستخدم
  - التحقق من وجود المستخدم قبل الإنشاء
  - التحقق من تعيين الدور قبل الإضافة

#### 2. إنشاء ملف اختبار الاتصال `test_db_connection.php`
- سكريبت بسيط للتحقق من اتصال قاعدة البيانات
- يمكن استخدامه لاختبار الاتصال قبل تشغيل reset_admin.php

### كيفية استخدام السكريبت المحدّث

#### الطريقة 1: تشغيل من سطر الأوامر (موصى به)
```powershell
cd C:\xampp\htdocs\projects\yemen-union-system
php reset_admin.php
```

#### الطريقة 2: اختبار الاتصال أولاً
```powershell
cd C:\xampp\htdocs\projects\yemen-union-system

```

### استكشاف الأخطاء

#### إذا ظهر خطأ "Class not found"
1. تأكد من تشغيل `composer install` في مجلد `backend`:
   ```powershell
   cd C:\xampp\htdocs\projects\yemen-union-system\backend
   composer install
   ```

2. تأكد من تشغيل `composer dump-autoload`:
   ```powershell
   cd C:\xampp\htdocs\projects\yemen-union-system\backend
   composer dump-autoload
   ```

#### إذا ظهر خطأ في الاتصال بقاعدة البيانات
1. تأكد من تشغيل XAMPP (MySQL/MariaDB)
2. تأكد من وجود قاعدة البيانات `yemen_union_db`
3. تحقق من بيانات الاعتماد في `backend/.env`:
   ```
   DB_HOST=localhost
   DB_PORT=3306
   DB_DATABASE=yemen_union_db
   DB_USERNAME=root
   DB_PASSWORD=
   ```

#### إذا ظهر خطأ "president role not found"
1. تأكد من إنشاء جدول `roles` وإدخال البيانات الأولية
2. قم بتشغيل ملف seed للأدوار إذا كان موجودًا

### الملاحظات الإضافية

- السكريبت الآن يتحقق من وجود المستخدم قبل الإنشاء
- إذا كان المستخدم موجودًا، يتم تحديث كلمة المرور فقط
- السكريبت يتحقق من تعيين الدور قبل الإضافة لتجنب التكرار
- جميع الرسائل باللغة العربية وواضحة
